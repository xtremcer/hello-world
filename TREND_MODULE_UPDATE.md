# 趋势维度（DK信号）模块更新说明

## ✅ 更新内容

已将 `modules/dimensions/trend.py` 模块从**直接获取 DK 信号**改为**自行计算 DK 信号**。

## 🔄 主要改动

### 原版实现
- 依赖 akshare 的 `stock_dk_signal` 接口
- 直接获取 DK 信号数据
- 接口不可用时需要手动配置

### 新版实现
- 使用 akshare 的 `stock_zh_a_hist` 接口获取历史数据
- 自行计算 EMA12、EMA26 等指标
- 根据技术指标判断 D 点和 K 点
- 计算 D 点后累计最大涨幅

## 📊 计算逻辑

### 模块 1：AkShare 数据获取
- **接口**: `ak.stock_zh_a_hist`
- **提取字段**: 日期、收盘价、最高价、成交量
- **处理**: 按日期升序排序，清洗数据

### 模块 2：指标计算
- **EMA12**: 12 日指数移动平均线
- **EMA26**: 26 日指数移动平均线
- **MAV5**: 5 日均量
- **金叉**: EMA12 上穿 EMA26
- **死叉**: EMA12 下穿 EMA26
- **5 日新高**: 收盘价为近 5 日最高
- **5 日新低**: 收盘价为近 5 日最低

### 模块 3：DK 信号标记

#### D 点（买点）4 条件
1. **金叉**: EMA12 上穿 EMA26
2. **5 日新高**: 收盘价为近 5 日最高
3. **成交量放大**: 成交量 > MAV5
4. **价格突破**: 收盘价 > EMA26

#### K 点（卖点）4 条件
1. **死叉**: EMA12 下穿 EMA26
2. **5 日新低**: 收盘价为近 5 日最低
3. **成交量放大**: 成交量 > MAV5
4. **价格跌破**: 收盘价 < EMA26

#### 去重规则
- 连续信号只保留第一个（手机 APP 规则）
- 避免连续多个 D 点或 K 点

### 模块 4：D 点后涨幅计算
- 找到所有 D 点
- 计算每个 D 点后所有交易日的最高价
- 累计最大涨幅公式：`((最高价 - D点价格) / D点价格) * 100`
- 保留 3 位小数（如 6.869%）

### 模块 5：结果输出
- **最新信号**: D / K / 无
- **最近 D 点后累计最大涨幅**: 百分比
- **历史信号统计**: D 点和 K 点的数量
- **近 20 日 DK 信号**: 详细列表
- **近 5 日指标详情**: EMA12、EMA26、成交量、MAV5

## 🎯 优势

1. **独立性**: 不依赖 akshare 的 DK 信号接口
2. **准确性**: 基于标准技术指标计算
3. **透明度**: 计算逻辑清晰可追溯
4. **完整性**: 包含历史信号统计和涨幅计算
5. **可扩展**: 可根据需要调整判断条件

## 🚀 使用方法

无需修改调用方式，直接使用即可：

```bash
~/stock/venv/bin/python /mnt/nas/stock/main.py
```

模块会自动：
1. 获取历史数据
2. 计算技术指标
3. 判断 DK 信号
4. 计算涨幅
5. 输出结果

## 🔧 参数调整

如需调整判断条件，可修改 `module3_mark_dk_signals` 函数：

```python
# D 点条件（当前 4 条件）
d_condition = (
    df['golden_cross'] &  # 1. 金叉
    df['new_high_5d'] &   # 2. 5 日新高
    (df['volume'] > df['mav5']) &  # 3. 成交量放大（> MAV5）
    (df['close'] > df['ema26'])  # 4. 价格突破 EMA26
)

# 可根据需要调整条件，例如：
# - 改变 EMA 周期（12/26）
# - 改变成交量放大倍数（> 1.5 * MAV5）
# - 添加或删除条件
```

## ⚠️ 注意事项

1. **数据要求**: 需要至少 30 天的历史数据
2. **计算时间**: 首次计算可能需要较长时间（获取历史数据）
3. **信号延迟**: 基于收盘数据，信号确认存在延迟
4. **参数优化**: 可根据实际效果调整 EMA 周期等参数
